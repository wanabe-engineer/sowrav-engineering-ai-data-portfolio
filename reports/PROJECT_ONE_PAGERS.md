# Project One-Pagers

Use these one-pagers as the polished explanation layer for GitHub, LinkedIn, professor emails, and job applications.

## 1. Biomedical Respiratory AI and Digital Stethoscope

**Portfolio role:** Flagship research project.

**Core idea:** Build a low-cost respiratory sound acquisition and AI analysis workflow that can support abnormal lung-sound screening research.

**Technical stack to emphasize:** Python, audio signal processing, MFCC/spectrogram features, CNN or 1D-CNN modeling, Raspberry Pi or embedded acquisition, clinical AI evaluation.

**Evidence in repo:** Research drafts, proposal documents, literature review files, project reports, and the new system architecture diagram.

**Reviewer value:** Shows a coherent research direction for Data Science, Biomedical Engineering, AI in Healthcare, and Engineering master's programs.

**Next build artifact:** A safe demo notebook using synthetic or public respiratory audio to show preprocessing, features, baseline model, and evaluation metrics.

## 2. VLSI Logic Gate Design Case Study

**Portfolio role:** Deep EEE fundamentals project.

**Core idea:** Present transistor-level logic design from Boolean function to schematic/layout artifact and verification notes.

**Technical stack to emphasize:** CMOS logic, truth tables, schematic/layout files, power-delay tradeoff, documentation.

**Evidence in repo:** `.sch`, `.MSK`, truth table markdown, and lab-report evidence.

**Reviewer value:** Shows engineering rigor and the ability to explain low-level technical artifacts clearly.

**Next build artifact:** Export schematic/layout screenshots and add them to the project page as visual proof.

## 3. Electrical Services Design CAD Portfolio

## 4. Power Systems Simulation and Lab Analysis

**Portfolio role:** Quantitative engineering analysis project.

**Core idea:** Convert lab work into reproducible calculations and plots around power factor, transformers, and AC system behavior.

**Technical stack to emphasize:** Python/MATLAB, phasors, real/reactive/apparent power, plots, experiment interpretation.

**Evidence in repo:** Power system lab report and power factor correction evidence.

**Reviewer value:** Shows comfort with numerical analysis and engineering interpretation.

**Next build artifact:** A notebook that calculates power quantities and creates clean visual plots from report values.

## 5. Control Systems Modeling and Response Analysis

**Portfolio role:** Systems modeling project.

**Core idea:** Rebuild control-system lab concepts as response-analysis notebooks with overshoot, settling time, steady-state error, and stability discussion.

**Technical stack to emphasize:** Transfer functions, step response, PID concepts, Python Control Systems Library or MATLAB.

**Evidence in repo:** Control-system reports and open-ended lab evidence.

**Reviewer value:** Shows modeling discipline and a bridge from EEE to data-driven performance analysis.

**Next build artifact:** A response dashboard comparing system parameters and metrics.
